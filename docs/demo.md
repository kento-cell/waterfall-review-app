# デモシナリオ

## 想定ストーリー

> 架空の SI プロジェクト「ABC 社 受注管理システム」のウォーターフォール開発において、PG 役の開発者がレビューにかかる時間を AI で短縮する。

| 役 | やること |
|---|---|
| **PM** | 観点ファイル (Excel/TXT) を整備して PG に渡す |
| **PG** | 設計書 (UI 機能概要書 / SS 構造設計書) を D&D 投入してレビュー指摘を取得 |
| **PM** | 横断レビュー (UI×SS) で仕様食い違いを最終確認 |

## 投入する検証データ

`samples/` に 5 ファイルを同梱:

| ファイル | 種別 | 目的 |
|---|---|---|
| `sample_UI_機能概要書.xlsx` | UI 機能概要書 | わざと「適切に処理する」「等」「必要に応じて」等の曖昧表現を含む |
| `sample_SS_構造設計書.xlsx` | SS 構造設計書 | PT 項目漏れ・条件分岐記述漏れを含む |
| `sample_観点_UI.xlsx` | 観点ファイル (UI 用) | PM が定義した必須観点 |
| `sample_観点_SS.xlsx` | 観点ファイル (SS 用) | PM が定義した必須観点 |
| `sample_観点_整合性.xlsx` | 観点ファイル (横断用) | UI×SS の対応関係チェック観点 |

**設計上わざと埋め込んだ欠陥 (9件):**

| # | 種別 | 欠陥内容 | 期待される指摘 |
|---|---|---|---|
| 1 | UI | 「適切に処理する」表記 | 曖昧表現 — 具体的な処理を明記 |
| 2 | UI | エラー時挙動が「等」で終わる | 列挙不完全 — 全パターン記述 |
| 3 | UI | 必須項目「権限」記述なし | 必須項目漏れ |
| 4 | UI | 入力チェックの境界値未定義 | 境界条件不明確 |
| 5 | SS | 条件分岐の「else」分記述なし | 分岐網羅性不足 |
| 6 | SS | PT 項目で正常系のみ、異常系なし | テスト観点漏れ |
| 7 | SS | DB 更新時の排他制御未記述 | 並行制御漏れ |
| 8 | UI×SS | UI に「CSV 出力」あり、SS に処理記述なし | 仕様食い違い |
| 9 | UI×SS | SS に「メール通知」あり、UI に画面記述なし | 仕様食い違い |

## 操作手順

### 1. ログイン

```
URL: http://localhost:3000/login
Email: demo@example.com
Pass : password123
```

### 2. プロジェクト作成

「+ 新規プロジェクト」→ 名前: `ABC社 受注管理システム`、フェーズ: 詳細設計をチェック

### 3. レビュー実行

プロジェクト詳細 → 「▶ AI レビュー実行」

```
① 工程選択       : 詳細設計
② 観点ファイル    : sample_観点_UI.xlsx を D&D
③ 成果物 (UI)    : sample_UI_機能概要書.xlsx を D&D
④ ▶ 実行
```

### 4. 結果確認

- 重要度別に件数表示 (🔴 高 / 🟡 中 / 🟢 低)
- 行クリックで指摘詳細ダイアログ
- 対応状況を変更 (未着手 → 対応中 → 完了 / 対象外)

### 5. PDF DL

「⬇ PDF 生成して DL」をクリック → 元 Excel の該当箇所に注釈オーバーレイされた PDF が DL される

### 6. 横断レビュー

別レビューとして:
```
② 観点ファイル    : sample_観点_整合性.xlsx
③ 成果物 (UI)    : sample_UI_機能概要書.xlsx
④ 成果物 (SS)    : sample_SS_構造設計書.xlsx
⑤ ▶ 実行
```

→ UI と SS の対応関係・食い違いを横断 AI で検出。

## Stub LLM の挙動

`LLM_PROVIDER=stub` の場合、固定で 2 件の指摘 (高 1 / 中 1) のみ返す。これは **動作確認** が目的で、実用評価には Anthropic API キー設定が必要。

## トラブルシュート

| 症状 | 対処 |
|---|---|
| ログイン失敗 (401) | ユーザ未作成。`docs/setup.md` の curl コマンドで作成 |
| ログイン失敗 (CORS) | `backend/.env` の `CORS_ORIGINS` に Frontend URL を追加 |
| PDF DL ボタンが「生成中」のまま | `backend/storage/pdf/` の権限確認 / Excel COM or LibreOffice インストール確認 |
| 指摘 0 件 | 観点ファイルが空 or `LLM_PROVIDER=stub` で観点と無関係に固定返却 |
